# Aquarium Environment Builder

Repo bootstrap pack for the Aquarium Environment Builder project.

This package contains the planning and execution documents prepared for a phased Codex-driven implementation workflow.

## Included files

- `INITIAL_SPEC.md` — full product specification and scope foundation
- `AGENTS.md` — recommended multi-agent structure and prompts
- `PHASE_1_PROMPT.md` — main Codex prompt and implementation guardrails for Phase 1
- `ACCEPTANCE_CHECKLIST.md` — completion checklist for Phase 1
- `SKILLS/` — reusable Codex skill templates for architecture, viewport work, schema protection, BOM metadata, phase reporting, and frontend regression review

## Intended use

1. Create or open your project repo.
2. Drop these files at the repo root.
3. Use `INITIAL_SPEC.md` as the product and architecture anchor.
4. Use `PHASE_1_PROMPT.md` as the main implementation prompt in Codex.
5. Use `AGENTS.md` to run specialist threads/worktrees.
6. Use the files under `SKILLS/` as reusable skills or internal guidance.
7. Use `ACCEPTANCE_CHECKLIST.md` before declaring Phase 1 done.

## Notes

- The contents were packaged without trimming down the planning material.
- This pack is intentionally focused on strong real-time 3D foundations, phased execution, and future support for printable/purchased/hybrid asset logic.
